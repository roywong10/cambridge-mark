# 标注 Annotation

**何时读:** 在 RM 工具里给作文加标注时。

**真源:** 本文件是标注用法的唯一真源。标注是**评分证据**,不是分数本身——分数存在 `S&A` / `C&S` 字段里(见 `paper-structure.md`)。

---

## 标注是证据,不是分数

标注层用来**展示判断背后的依据**,让分数可追溯。它不替代评分决策,也不驱动最终分。安全流程:

```text
读 response → 标出有用的证据 → 决定 S&A 和 C&S → 录分 → 核对总分 → 保存
```

## 标记类型

### 正向证据

- `Tick`:有效的点、清晰的想法、或有效的措辞。
- `DET`:有用的细节、展开、或支撑材料。

### 问题

- `?`:意思不清或表达令人困惑。
- 波浪线(`H Wavy`):局部语言问题、别扭措辞、或不清的构造。
- `Highlight`:字词或用词问题(也用于标出支撑某个 band 的证据)。
- `^`:遗漏 / 缺失 / 需插入处。

### Housekeeping

- `SEEN`:用在空白页,或已读但未另加标注的页 / 段。

## RM 工具侧实际可见的标注

在已批脚本里出现过的标注:`SEEN`、`Highlight`、`H Wavy`、`^`。其余(`Tick` / `DET` / `?` / 波浪线)按上面的语义使用。

## 纪律

- 标注**稀疏但有代表性**,不要逐行标。
- 用 `SEEN` 表示某页 / 段已读。
- 用 `Highlight` 标支撑所给 `S&A` / `C&S` band 的证据。
- 用 `H Wavy` 标值得注意的语言问题、别扭表达、或影响评分的写作质量处。
- 用 `^` 标缺失 / 插入 / 影响判断的空缺处。
- 当分数取决于一个 **judgement call** 时,正负标记都放几个,让档位有据可依。
